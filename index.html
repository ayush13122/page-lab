How It Works: Page Replacement Algorithms

A quick refresher on the logic applied by each simulation method.

FIFO (First-In, First-Out)
Simple Queue Logic

The operating system keeps track of all pages in memory in a queue. The page that has been in memory the longest is replaced.

  • Easy to implement and understand.
  • Ignores the actual usage frequency of the pages.
  • Suffers from Belady's Anomaly (faults can increase with more frames).
LRU (Least Recently Used)
Temporal Locality

Pages that have been heavily used in the past will probably be used again soon. The page that has not been used for the longest period of time is replaced.

  • Very good performance in practice.
  • Requires complex hardware support (counters, timestamps) to track usage.
  • High implementation overhead.
Optimal (MIN)
Perfect Prediction

This is a theoretical algorithm. The page chosen for replacement is the one that will not be used for the longest period of time in the future.

  • Yields the lowest possible page fault rate.
  • Cannot be implemented in a real operating system.
  • Used as a benchmark to compare real-world algorithms.
